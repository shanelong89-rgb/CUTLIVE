Technical Implementation
Component
Details
Credit table
New table in Supabase: user_credits. Fields: user_id, balance, total_earned, total_spent, last_updated.
Credit ledger
New table: credit_transactions. Fields: id, user_id, amount, type (submission_approved / invite_signup / future: event_purchase), reference_id, created_at.
Balance calculation
Sum of all credit_transactions for user_id. Display in account page.
Invite tracking
New table: invite_links. Fields: id, user_id, invite_code (unique), created_at, times_used. New table: invite_signups. Fields: id, invite_code, invited_user_id, inviter_user_id, verified_at.
Credit trigger — submissions
When editorial review status changes to "approved", system checks if credit already issued for this submission. If not, inserts credit_transaction (+50) and updates user_credits balance.
Credit trigger — invites
When invited user verifies email, system checks invite_signups table. If valid invite and no duplicate, inserts credit_transaction (+25) for both inviter and invited user.
Fraud prevention
One credit per submission (tracked by submission_id). One credit per unique invited email (no duplicates). Invite link abuse detection: if same IP/device creates multiple accounts via invite, flag for review.
Credit expiry
No expiry for now. Credits are permanent until used.

3. Invite System
How It Works
Every signed-up user gets a unique invite link. They share it. When someone signs up via that link and verifies their email, both the inviter and the invited user get HK$25 credit.
