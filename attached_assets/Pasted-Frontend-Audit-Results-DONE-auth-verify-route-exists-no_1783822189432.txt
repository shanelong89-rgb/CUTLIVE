🟡 TYPE MISMATCH: category_affinities as string[] vs object
supabase.ts line 811 types it as string[] but the DB stores it as {"art": 12, "music": 6}. The (data as any).category_affinities ?? [] cast on an object won't become [] — it stays as the object. Then normalizeCategories(raw.category_affinities) calls .map() on an object, which returns undefined for each iteration (.map on objects returns empty). This means Settings won't show any categories. Fix: use Object.keys(raw.category_affinities) to get the array of category IDs.
