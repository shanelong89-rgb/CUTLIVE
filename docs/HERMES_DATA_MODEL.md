# CULTIVE — Hermes Data Model

This document is the canonical specification for how Hermes (the scraper/enrichment agent) must populate event data so it appears correctly in the CULTIVE frontend. Mismatches in `category` or `tags` cause events to be invisible in filters.

---

## Events Table Schema

All events live in `public.events` in the Supabase project (`qmjdqldmpmeguuyepbsw`).

| Column | Type | Required | Notes |
|---|---|---|---|
| `id` | `text` PK | auto | Generated by the app as `event_<timestamp36>_<rand>` |
| `title` | `text` | **required** | Event name |
| `date` | `text` | **required** | Format: `YYYY-MM-DD` (e.g. `2026-06-15`) |
| `date_end` | `text` | optional | Same format as `date`. Omit if single-day event |
| `time` | `text` | recommended | Human-readable, e.g. `8:00 PM`, `10:30 PM`, `11:00 PM` |
| `venue` | `text` | **required** | Full venue name, e.g. `Mihn Club, Wan Chai` |
| `district` | `text` | recommended | Hong Kong district, e.g. `Central`, `Wan Chai`, `Causeway Bay` |
| `category` | `text` | **required** | Must match the canonical list — see below |
| `tags` | `text[]` | **required** | Must be a **non-empty array** of values from the canonical tag list — see below |
| `price` | `text` | recommended | e.g. `$200`, `Free`, `Free entry`, `From $150` |
| `description` | `text` | recommended | Plain text or HTML (Tiptap-compatible) |
| `image` | `text` | recommended | Public HTTPS image URL |
| `is_exclusive` | `boolean` | optional | `true` for members-only events. Default `false` |
| `ticket_url` | `text` | optional | Must be a valid `https://` URL |
| `source_url` | `text` | optional | Original source URL (e.g. Instagram post, venue page) |
| `rsvp_enabled` | `boolean` | optional | `true` to show an RSVP button on the event page |
| `submitted_by` | `text` | optional | Display name of who curated the event |
| `metadata` | `jsonb` | optional | Freeform key-value store for Hermes internal data — see below |

## Canonical Category Values

The `category` field must be **exactly one** of these strings (case-sensitive):

| Category | Primary tag |
|---|---|
| `Music` | `music` |
| `Arts` | `art` |
| `Nightlife` | `nightlife` |
| `Food` | `food` |
| `Wellness` | `wellness` |
| `Market` | `market` |
| `Workshops` | `workshops` |
| `Other` | _(none)_ |

> **Rule**: `category` must match the primary tag for the event. If you set `category: "Food"`, the `tags` array must include `"food"`. Mismatches make the event disappear from the filter.

---

## Canonical Tag Values

The `tags` field must be a **non-empty `text[]`** using only values from this list:

| Tag ID | Label | Notes |
|---|---|---|
| `music` | Music | Live music, concerts, DJ sets, jazz, bands |
| `electronic` | Electronic | Electronic/techno/house — use alongside `music` |
| `nightlife` | Nightlife | Club nights, late-night events (after 17:00), bar events |
| `art` | Art | Galleries, exhibitions, street art, visual arts |
| `market` | Market | Night markets, craft fairs, pop-up markets |
| `food` | Food | Dining events, food festivals, supper clubs, cocktail bars |
| `wellness` | Wellness | Yoga, meditation, fitness, wellbeing |
| `workshops` | Workshops | Classes, skill sessions, creative workshops |
| `community` | Community | Networking, talks, cultural gatherings |

### Tag rules

- **`tags` must NEVER be empty** (`[]`). A missing or empty `tags` field makes the event unfilterable.
- Multiple tags are allowed and encouraged (e.g. `["music", "electronic", "nightlife"]` for a techno night).
- The **primary tag** (the one matching `category`) should be the **first element** in the array.
- The auto-approval logic in `supabase.ts > approveSubmission` will append `nightlife` automatically if the event time is 17:00 or later. You do not need to add it manually for late-night events, but it is safe to include it.

### Category ↔ Tag mapping

```
Music      → tags must include "music"
Arts       → tags must include "art"        ← NOT "arts"
Nightlife  → tags must include "nightlife"
Food       → tags must include "food"
Wellness   → tags must include "wellness"
Market     → tags must include "market"
Workshops  → tags must include "workshops"
Other      → no required primary tag
```

## Date & Time Formats

| Field | Format | Examples |
|---|---|---|
| `date` | `YYYY-MM-DD` | `2026-06-15`, `2026-12-31` |
| `date_end` | `YYYY-MM-DD` | `2026-06-17` (for multi-day events) |
| `time` | Human-readable | `8:00 PM`, `10:30 PM`, `11:00 PM`, `7:30 AM` |

- Use **12-hour format with AM/PM** for `time` — this is displayed directly to users.
- Do **not** use ISO 8601 or 24-hour format for `time`.
- Dates stored as text (not `timestamptz`) — the frontend sorts and displays them as-is.

## Submissions Table (`scraped_data` field)

When Hermes enriches an Instagram submission it writes extracted fields into `scraped_data` (a `jsonb` column on `public.submissions`). The `approveSubmission` function reads these as fallbacks when the primary fields are absent.

### Recognised `scraped_data` keys

| Key | Type | Used as fallback for |
|---|---|---|
| `extracted_title` | `string` | `title` |
| `extracted_date` | `string` | `date` (must be `YYYY-MM-DD`) |
| `extracted_time` | `string` | `time` |
| `extracted_venue` | `string` | `venue` |
| `extracted_description` | `string` | `description` |
| `extracted_image` | `string` | `image` |
| `extracted_category` | `string` | `category` |
| `extracted_price` | `string` | `price` |
| `extracted_district` | `string` | `district` |

> **Important**: Hermes should **also** populate the top-level submission fields (`title`, `date`, `venue`, `category`, `tags`, etc.) directly whenever possible. The `scraped_data` keys are fallbacks only — they are used if the top-level field is empty or null.

### `metadata` field (events & submissions)

Use `metadata` for Hermes-internal tracking data that doesn't belong in a dedicated column:

```json
{
  "hermes_version": "1.2.0",
  "source_platform": "instagram",
  "enriched_at": "2026-05-23T10:00:00Z",
  "confidence": 0.92,
  "raw_caption_snippet": "…first 200 chars of caption…"
}
```

This field is not displayed to users; it is for audit and debugging.

## Worked Example — Correctly-Formed Event

A food/dining event scraped from Instagram and published via the admin approval flow:

```json
{
  "id": "event_lp4x2k_abc123",
  "title": "Adele's Homecoming Dinner",
  "date": "2026-06-20",
  "date_end": null,
  "time": "7:00 PM",
  "venue": "Adele's, Sheung Wan",
  "district": "Sheung Wan",
  "category": "Food",
  "tags": ["food"],
  "price": "$450",
  "description": "<p>A five-course homecoming dinner celebrating Cantonese flavours reimagined by chef Adele Leung. Advance booking required.</p>",
  "image": "https://example.com/adeleshomecoming.jpg",
  "is_exclusive": false,
  "ticket_url": "https://adeleshk.com/book",
  "source_url": "https://www.instagram.com/p/AbCdEfGhIj/",
  "rsvp_enabled": false,
  "submitted_by": "Hermes",
  "metadata": {
    "hermes_version": "1.2.0",
    "source_platform": "instagram",
    "enriched_at": "2026-05-23T08:14:00Z"
  }
}
```

### Common mistakes to avoid

| Mistake | Effect | Fix |
|---|---|---|
| `category: "Food"` but `tags: []` | Event invisible in Food filter | Always populate `tags` from the canonical list |
| `category: "Arts"` but `tags: ["arts"]` | Event invisible in Art filter | Use `"art"` (no trailing s) |
| `category: "Music"` for a food event | Admin must correct manually | Derive category from event content accurately |
| `time: "20:00"` (24h format) | Displays incorrectly to users | Use `"8:00 PM"` |
| `date: "June 20 2026"` | Date display and sort breaks | Use `"2026-06-20"` |
| Missing `tags` key entirely | Same as empty — event unfilterable | Always include `tags` as a non-empty array |

## Admin Remediation

After correcting Hermes to write the right fields going forward, existing mismatched events must be fixed manually:

1. Open the CULTIVE admin panel → Events tab
2. Find the mismatched event (e.g. "Adele's Homecoming")
3. The editor will now auto-derive the correct Category and Tags from the event data
4. Confirm the Category dropdown and Tag pills are correct, then click **Save**
5. The corrected `category` and `tags` are written back to Supabase

See `artifacts/cultive/src/data/events.ts` — `CATEGORY_TAG_MAP` and `TAG_CATEGORY_MAP` — for the canonical mapping used by both the frontend filter and the admin editor.
